Repix provides a self-hosted alternative to ImageKit. This guide maps ImageKit transformation parameters to Repix equivalents.

## URL Structure

| Platform   | URL Pattern                                  |
| ---------- | -------------------------------------------- |
| **ImageKit** | `https://ik.imagekit.io/{id}/tr:{params}/{path}` |
| **Repix**    | `https://your-repix.com/{params}/{path-without-https}` |

Repix uses `key=value` parameters separated by commas. The image path is the URL **without** `https://` (Repix adds it automatically).

## Parameter Mapping

ImageKit uses hyphen-separated parameters in a `tr:` block (e.g. `tr:w-400,h-300,cm-extract`). Repix uses `key=value` with commas.

| ImageKit | Repix | Notes |
| -------- | ----- | ----- |
| `w-400` | `w=400` | Width |
| `h-300` | `h=300` | Height |
| `cm-extract` / fill | `fit=cover` | Crop to fill |
| `cm-fit` | `fit=contain` | Fit within dimensions |
| `cm-scale` | `w=W` or `h=H` | Scale proportionally |
| `cm-pad_resize` | `fit=pad` | Pad with background |
| `fo-webp` | `f=webp` | Output format |
| `fo-jpg` | `f=jpeg` | Output format |
| `q-85` | `q=85` | Quality 1–100 |
| `rt-90` | `rotate=90` | Rotation degrees |
| `cm-flip` (or flip params) | `flip=h` or `flip=v` | Flip direction |

## Example Conversions

**ImageKit:**
```http
https://ik.imagekit.io/demo/tr:w-400,h-300,cm-extract,fo-webp,q-85/example.jpg
```

**Repix:**
```http
https://your-repix.com/w=400,h=300,fit=cover,f=webp,q=85/ik.imagekit.io/demo/example.jpg
```

If ImageKit is proxying from another origin, use the actual source URL in Repix:

```http
https://your-repix.com/w=400,h=300,fit=cover/cdn.example.com/images/example.jpg
```

## Migration Steps

1. **Deploy Repix** — Use [Docker](/deployment/render) or your preferred platform. Point a subdomain (e.g. `img.yourdomain.com`) to it.
2. **Configure presets** — Add presets that match your ImageKit named transforms.
3. **Update image URLs** — Replace the ImageKit base URL with Repix in your app (e.g. via a helper or env variable).
4. **Source images** — Ensure images are publicly accessible over HTTPS. If migrating from ImageKit storage, move assets to S3/R2 first, then point Repix at those URLs.
5. **Run in parallel (optional)** — Use a feature flag to switch between ImageKit and Repix during migration.

## Features Not Supported

Repix focuses on core image transformations. AI cropping, overlays, video, and client hints (DPR auto) are not supported. Use `fit=cover` for cropping and `blur=20` for placeholder effects.
